DBMS
1. Introduction to DBMS

Database: Organized collection of data.

DBMS: Software to manage databases.

Examples: MySQL, Oracle, PostgreSQL, MongoDB

2. Characteristics of DBMS
  • Data Redundancy Control
  • Data Independence
  • Efficient Query Processing
  • Concurrent Access
  • Security and Authorization
  • Data Integrity
  • Backup and Recovery
3. Components of DBMS
  • Hardware
  • Software
  • Data (with metadata)
  • Users: DBA, Programmers, End Users
  • Procedures
4. Database Models
  • Hierarchical Model
  • Network Model
  • Relational Model
  • Object-Oriented Model
  • Document/NoSQL Model
5. DBMS Architecture

6. Data Independence
  • Logical: Schema changes don't affect views.
  • Physical: Storage changes don't affect schema.
7. Database Languages
  • DDL: CREATE, ALTER, DROP
  • DML: SELECT, INSERT, UPDATE, DELETE
  • DCL: GRANT, REVOKE
  • TCL: COMMIT, ROLLBACK, SAVEPOINT
8. Keys in DBMS
  • Primary Key
  • Candidate Key
  • Foreign Key
  • Super Key
  • Composite Key
9. Normalization

Eliminates redundancy and maintains data integrity.

  • 1NF: Atomic values
  • 2NF: No partial dependency
  • 3NF: No transitive dependency
  • BCNF: Stronger 3NF
10. SQL Basics
SELECT * FROM table;
INSERT INTO table VALUES (...);
UPDATE table SET column = value WHERE ...;
DELETE FROM table WHERE ...;
